Атрибуты директорий и файлов

Пример: rwx r-x r-x

Параметры атрибутов состоят из девяти знаков, которые на самом деле представляют собой три группы по три символа. Каждая такая группа определяет права для какой-либо из трех категорий пользователей

  • первая группа - права "хозяина"
  • вторая группа - права "группы особо допущенных"
  • третья группа - права для "всех остальных"

Смысл отдельных символов в каждой такой группе "прав" одинаков для всех трех категорий пользователей, поэтому можно подробнее рассматривать любую такую группу, не уточняя - для какой категории пользователей она предназначена.

Для файлов

Первый символ, обозначается буквой "r" (read), и означает, что пользователю, подпадающему под соответствующую категорию, разрешается читать содержимое этого файла. То есть он может посмотреть содержимое файла, а также скопировать этот файл. Кстати, это не означает, что пользователь сможет запустить его на выполнение (если это программа).

Второй символ, обозначается буквой "w" (write) и разрешает писать в файл. То есть пользователь сможет изменить содержимое файла (например, каким-нибудь редактором), дописать что-нибудь в конец или стереть все содержимое. Обратите внимание, этот символ еще не дает право удалить сам файл из директории или изменить ему название (это определяется правами на саму директорию), но дает возможность сделать этот файл пустым (нулевой длины) или скопировать в него содержимое другого файла (и тем самым "подменить" его).

Третий символ, обозначается буквой "x" (eXecute), позволяет запустить на выполнение этот файл, если он представляет собой программу или командный файл. Обратите внимание, что это также основной признак, по которому система догадывается о "запускаемости" этого файла.

Для директорий

Первый символ ("r") разрешает читать оглавление этой директории, то есть список файлов и поддиректорий, находящихся в ней. Однако, этот символ еще не дает возможность получить доступ к содержимому, то есть читать/запускать/изменять файлы, даже если "права доступа" установленные на самих файлах это позволяют. Поэтому, само по себе "право чтения" директории практически бесполезно и этот символ, как правило ставиться только вместе с символом "x".

Второй символ "w", установленный на директории, позволяет изменять оглавление директории. То есть, разрешает создавать новые файлы (или копировать другие файлы в эту директорию), менять названия файлов и удалять файлы. Если права на директорию не позволяют пользователю удалить файл, находящийся в ней (нет символа "w") это не означает, что пользователь не сможет "удалить содержимое" файла (например, "вытерев" все в нем редактором). С другой стороны, если пользователь имеет право менять содержимое директории, он сможет удалить или переименовать любой, находящийся в ней файл, даже если права на самом файле не позволяют ему не то что писать в файл, но и читать его.

Третий символ - "x". Для директорий он как раз означает, что пользователь может получит доступ к "компонентам", то есть отдельным файлам и поддиректориям. Только при наличии это символа, система разрешит войти в эту директорию и выполнить какое-нибудь действие с файлом, если сами файлы ("права доступа" на них) это позволят. Обратите внимание, что если даже внутренние поддиректории имеют "нормальные" права для какой-то категории пользователей, а вышестоящая директория - нет (отсутствует бит "x"), то этим пользователям не удастся "занырнуть" в поддиректории, минуя вышестоящую. Система проверяет полный "путь" до конечной директории или файла (например, /usr/share/misc/fonts) и, если хотя бы один из компонентов этого пути не имеет соответствующего символа, то пользователю будет отказано в доступе.

Поскольку, все символы доступа это двоичные биты, то можно указать сразу число (в восьмеричном виде). Таким образом, вам надо просто указать три цифры, каждая из которых описывает отдельную категорию пользователей (точнее - права для этой категории). Например, rwx r-x r-x можно представить в восьмеричном виде как 755.

Всегда r = 4,w = 2,x = 1. Из вышеприведенного примера получается, для владельца rwx = 4+2+1 = 7, для допущенной группы пользователей r-x = 4 + 1 = 5, и для остальных пользователей r-x = 4 + 1 = 5. В итоге и получаются права доступа в цифровом представлении 755 = rwx r-x r-x.
